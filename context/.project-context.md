# Project Context — StructFlow

## What it is

**StructFlow** is a Chrome (Manifest V3) **side-panel extension** that combines:

- a **JSON viewer** (interactive tree + pretty-printed, syntax-highlighted text), and
- a **multi-language code formatter / note surface** (Markdown, Plain Text,
  TypeScript, JavaScript, JSON, HTML, CSS, SQL), and
- a **saved-entries library** with projects/folders, search, and
  copy/save/export/re-open, and
- a **code snapshot** exporter (CodeSnap-style PNG of the formatted output).

It runs as a real unpacked MV3 extension and also previews as a plain web app
during development (same code, IndexedDB works in both contexts).

## Who/why

For developers who frequently inspect and clean up structured data and code
snippets while browsing — paste anything into the side panel, format it, explore
it, and keep a searchable library of saved snippets organized by project/folder.
Because **Markdown** and **Plain Text** are first-class first-choice languages, it
also doubles as a lightweight **note-taker** that lives in the browser side panel.

## Core features

- **Format** with VS Code-style defaults plus extra options (tab width,
  semicolons, single/double quotes, print width, wrap, etc.).
- **Note-taking**: Markdown (default) and Plain Text are first-class languages;
  Plain Text is a pure pass-through that is never reformatted.
- **Markdown preview**: rendered preview toggles against the source.
- **JSON validity** indicator (Valid/Invalid badge).
- **Two JSON views**: collapsible tree (with key/value search, copy-path,
  copy-value, type color-coding) and highlighted text view with line numbers.
- **In-page JSON viewer**: a content script takes over raw `.json` pages /
  JSON responses, rendering a Formatted (default) / Tree / Raw view with its own
  Editor & JSON theme picker and search.
- **Syntax themes**: multiple editor/JSON color themes (VS Code, GitHub,
  Monokai, Dracula, Nord, Solarized…) applied independently of app chrome.
- **Code snapshot**: export the formatted output as a CodeSnap-style PNG —
  windowed card, traffic lights, filename, backdrop choices, padding, line
  numbers, Copy PNG / Download PNG.
- **Library**: save formatted output to IndexedDB, organize into projects/folders
  (per-folder ⋯ menu: rename, add new item, editable color swatches, delete),
  full-text search across title + content, re-open / copy / export / delete.
  "Add new item" jumps to the Formatter pre-targeting that folder.
- **Bulk export**: an Archive action exports selected (or all) folders as a single
  `.zip` — files foldered by project plus a `manifest.json` for full restore.
- **Theme**: dark + light, persisted, applied to both viewer and panel.
- **Support**: header popover with configurable GitHub / Buy Me a Coffee /
  Sponsor links (`src/lib/support-links.ts`).

## Conventions

- Language: **TypeScript**, React function components + hooks.
- Styling: **Tailwind (v4, `@tailwindcss/vite`)** with CSS variables/design tokens
  in `src/index.css`; theme switched via a class on the root element.
- No server / no network calls — fully offline. All persistence is **IndexedDB**.
- Formatting is done client-side via Prettier standalone + sql-formatter.
- Bulk export builds the `.zip` client-side via `fflate` (no upload).
- Brand assets are generated from `src/assets/logo-source.png` via
  `scripts/gen-icons.cjs` (sharp) → trimmed `logo.png` + icon sizes + favicon.

## Non-goals (current version)

- No accounts, sync, or backend.
- No auto-capture of in-page network JSON beyond the raw-page takeover yet.
- SQL is spacing/indentation cleanup only (not a full dialect-aware reformat).

## Build / run

- Dev: `pnpm dev` (Vite).
- Build: `pnpm build` → `dist/`.
- Install: `chrome://extensions` → Developer mode → Load unpacked → `dist/`.
